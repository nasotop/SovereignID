# SovereignID — Guía de Dockerización Moderna\n\n## Resumen Ejecutivo\n\nEsta guía documenta la estrategia de **containerización moderna** para el monorepo SovereignID:\n- Backend: ASP.NET Core 10 (auth-api)\n- Frontend: Angular 22 + Nginx Reverse Proxy (web)\n- Orquestación: Docker Compose 3.8+\n\n### Características Implementadas (Best Practices 2024-2026)\n\n✅ **Health Checks** — Monitoreo automático de salud de contenedores  \n✅ **Resource Limits** — CPU y memoria limitadas para evitar resource exhaustion  \n✅ **Security Headers** — CORS, CSP, X-Frame-Options en Nginx  \n✅ **Non-root Users** — Contenedores ejecutados como usuario `nginx`  \n✅ **Gzip Compression** — Compresión automática de assets  \n✅ **Logging Rotativo** — Logs sin crecimiento infinito (max-size, max-file)  \n✅ **Build Optimization** — Multi-stage, .dockerignore, layer caching  \n✅ **Labels & Metadata** — Información de imagen en los contenedores  \n✅ **Security** — no-new-privileges, dotfile protection  \n✅ **Proxy Headers** — X-Real-IP, X-Forwarded-* correctos  \n\n---\n\n## 1. Estructura de Archivos\n\n```\nSovereignID/\n├── docker-compose.yml              # Producción\n├── docker-compose.override.yml     # Desarrollo (auto-cargado)\n├── .dockerignore                   # Optimización de build\n├── .env.example                    # Variables de entorno\n├── Makefile                        # Comandos útiles\n├── README.Docker.md                # Este archivo\n│\n├── src/auth/\n│   └── Auth.Api/Dockerfile         # Backend .NET 10 (ya existente)\n│\n└── src/web/\n    ├── Dockerfile                  # Frontend Angular + Nginx\n    ├── nginx.conf                  # Configuración Nginx moderna\n    └── .dockerignore               # Optimización web build\n```\n\n---\n\n## 2. Configuración del Entorno\n\n### 2.1 Crear archivo `.env` para desarrollo\n\n```bash\ncp .env.example .env\n```\n\nEditar `.env` con tus valores (especialmente `AUTH_JWT_SIGNING_KEY`):\n\n```bash\n# Generar clave JWT segura\nmake generate-jwt-key\n# Resultado: ejemplo: AbC123dEfGhIjKlMnOpQrStUvWxYz+/=\n# Copiar y pegar en .env como AUTH_JWT_SIGNING_KEY=AbC123dEfGhIjKlMnOpQrStUvWxYz+/=\n```\n\n### 2.2 Validar docker-compose.yml\n\n```bash\nmake validate\n# O manualmente:\ndocker-compose config\n```\n\n---\n\n## 3. Comandos Principales\n\n### Development\n\n```bash\n# Construir imágenes\nmake build\n\n# Levantar servicios (con logs)\nmake up\n\n# Ver logs en vivo\nmake logs\nmake logs-api   # Solo auth-api\nmake logs-web   # Solo web\n\n# Reiniciar servicios\nmake restart\n\n# Detener\nmake down\n\n# Limpiar todo\nmake clean\n```\n\n### Testing & Debugging\n\n```bash\n# Ejecutar tests del backend\nmake test\n\n# Acceder a shell del contenedor\nmake shell-api  # Auth API\nmake shell-web  # Nginx/Web\n\n# Ver estado de servicios\nmake status\n```\n\n### Producción\n\n```bash\n# Construir para producción\nmake build-prod\n\n# Pushear a registry (requiere configuración)\nmake push-images\n```\n\n---\n\n## 4. URLs de Acceso Local\n\n| Servicio | URL | Descripción |\n|----------|-----|-------------|\n| **Frontend** | http://localhost:4200 | Angular app |\n| **Backend API** | http://localhost:8080 | Auth API directa |\n| **Auth Nonce** | http://localhost:4200/auth/nonce | Via Nginx proxy |\n| **Auth Verify** | POST http://localhost:4200/auth/verify | Via Nginx proxy |\n| **Nginx Health** | http://localhost:4200/nginx-health | Status check |\n\n---\n\n## 5. Arquitectura de Servicios\n\n```\n┌─────────────────────────────────────────────────────────────┐\n│ Cliente (Browser)                                           │\n│   - Angular app en http://localhost:4200                   │\n│   - Llamadas AJAX/Fetch a /auth/*                          │\n└────────────────────┬────────────────────────────────────────┘\n                     │\n        ┌────────────┴────────────┐\n        ↓                         ↓\n   Nginx Container         Auth-API Container\n   (Port 80→4200)          (Port 8080)\n   - Serve static assets   - .NET 10 runtime\n   - Proxy /auth/* requests - SIWE verification\n   - Security headers      - JWT issuance\n   - Gzip compression      - Health check endpoint\n   - Cache assets\n        │                         │\n        └────────────┬────────────┘\n                     ↓\n         sovereign-net (Docker Network)\n         172.25.0.0/16\n```\n\n---\n\n## 6. Health Checks\n\n### Auth API Health\n\n```bash\ncurl -f http://localhost:8080/health\n# Respuesta esperada: 200 OK (el endpoint debe implementarse en Backend)\n```\n\n### Nginx Health\n\n```bash\ncurl -f http://localhost:4200/nginx-health\n# Respuesta: \"healthy\\n\"\n```\n\n### Ver estado en Docker\n\n```bash\ndocker-compose ps\n# Estado esperado:\n# NAME                    STATE\n# sovereignid-auth        Up (healthy)\n# sovereignid-web         Up (healthy)\n```\n\n---\n\n## 7. Variables de Entorno Críticas\n\n| Variable | Tipo | Requerido | Nota |\n|----------|------|-----------|------|\n| `ASPNETCORE_ENVIRONMENT` | string | ✅ | Development o Production |\n| `AUTH_JWT_SIGNING_KEY` | string (≥32 bytes) | ✅ | Clave simétrica HS256 |\n| `Auth__AllowedChainId` | number | ⭕ | 11155111 (Sepolia) o 1 (Mainnet) |\n| `Auth__JwtIssuer` | string | ⭕ | sovereignid-auth (por defecto) |\n| `Auth__JwtAudience` | string | ⭕ | sovereignid-clients (por defecto) |\n| `API_BASE_URL` | string | ⚠️ | Frontend: http://auth-api:8080 (Docker) |\n\n---\n\n## 8. Seguridad\n\n### Headers de Seguridad (Nginx)\n\n```\nX-Frame-Options: SAMEORIGIN                    # Evita clickjacking\nX-Content-Type-Options: nosniff                # Evita MIME sniffing\nX-XSS-Protection: 1; mode=block                # Protección XSS\nContent-Security-Policy: ...                   # Control de recursos\nReferrer-Policy: strict-origin-when-cross-origin\nPermissions-Policy: ...                        # Control de API\n```\n\n### Non-root User\n\n- Web container: `USER nginx` (alpine nginx user)\n- Backend: Automático en .NET runtime\n\n### CORS en Backend (Próxima Implementación)\n\nAgregar a `Program.cs` en Auth.Api:\n\n```csharp\nbuilder.Services.AddCors(options =>\n{\n    options.AddPolicy(\"Angular\", policy =>\n        policy\n            .WithOrigins(\"http://localhost:4200\", \"http://localhost:3000\")\n            .AllowAnyMethod()\n            .AllowAnyHeader()\n            .AllowCredentials()\n    );\n});\n\napp.UseCors(\"Angular\");\n```\n\n---\n\n## 9. Logs y Debugging\n\n### Ver logs en tiempo real\n\n```bash\n# Todos los servicios\ndocker-compose logs -f\n\n# Solo auth-api\ndocker-compose logs -f auth-api\n\n# Últimas 100 líneas\ndocker-compose logs --tail=100 auth-api\n```\n\n### Archivos de log en contenedor\n\n**Nginx** (en container):\n- `/var/log/nginx/access.log`\n- `/var/log/nginx/error.log`\n\n**Auth API** (stdout del .NET runtime)\n\n### Log Rotation (json-file driver)\n\n- **max-size**: 10MB por archivo\n- **max-file**: 3 archivos máximo\n- Automático: Sin intervención manual\n\n---\n\n## 10. Performance y Recursos\n\n### Límites Actuales (Recomendado para Dev)\n\n| Servicio | CPU | Memoria | Caso de Uso |\n|----------|-----|---------|-------------|\n| auth-api | 0.5 | 512M | Desarrollo local |\n| web | 0.5 | 256M | Desarrollo local |\n\n### Ajustar recursos\n\nEn `docker-compose.yml`:\n\n```yaml\nresources:\n  limits:\n    cpus: '1'          # 1 CPU completo\n    memory: 1G         # 1 GB RAM\n  reservations:\n    cpus: '0.5'\n    memory: 512M\n```\n\n---\n\n## 11. Troubleshooting\n\n### Problema: \"Connection refused\" entre contenedores\n\n**Causa**: Usando `localhost` en lugar del nombre del servicio  \n**Solución**: Usar nombre del contenedor DNS (`auth-api`, `web`)\n\n```diff\n- proxy_pass http://localhost:8080\n+ proxy_pass http://auth-api:8080\n```\n\n### Problema: Health check failing\n\n```bash\n# Revisar logs\ndocker-compose logs auth-api\n\n# Health check manual\ndocker-compose exec auth-api curl -f http://localhost:8080/health\n\n# Aumentar start_period si el servicio tarda en iniciar\n# En docker-compose.yml: start_period: 60s\n```\n\n### Problema: CORS bloqueado en Angular\n\n**Causa**: Backend no configura `Access-Control-Allow-Origin`  \n**Solución**: Implementar CORS en Auth.Api Program.cs (ver sección 8)\n\n### Problema: Archivos compilados no actualizados\n\n```bash\n# Reconstruir sin cache\nmake clean\nmake build\nmake up\n```\n\n---\n\n## 12. Deployment a Producción\n\n### Pre-requisitos\n\n1. Generar `AUTH_JWT_SIGNING_KEY` segura:\n   ```bash\n   make generate-jwt-key\n   ```\n\n2. Cambiar `ASPNETCORE_ENVIRONMENT=Production`\n\n3. Usar Docker secrets o gestores de secretos:\n   ```bash\n   echo \"mi-secreto-real\" | docker secret create jwt_key -\n   ```\n\n4. Configurar CORS con dominios reales\n\n5. Usar HTTPS con certificados válidos\n\n### Docker Swarm o Kubernetes\n\nPara orquestación en producción, considerar:\n- **Docker Swarm**: Standalone, version simplificada de `docker-compose`\n- **Kubernetes**: Escalabilidad, auto-healing, rolling updates\n\n---\n\n## 13.
